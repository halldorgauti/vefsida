Mín verkefni

Hér má sjá þau verkefni sem ég hef unnið að á undanförnum árum.

Hönnun á vefsíðu

Hér mun ég fara yfir hvernig ég hannaði mína eigin vefsíðu. Þessi síða var búin til með þeim tilgangi að geyma helstu verkefni sem framkvæmd eru í áfangaum framleiðsluferli í Háskóla Íslands. Einnig mun ég nýta tækifærið og gera mitt besta við að uppfæra öllum helstu verkefnum mínum á þessa síðu. Síðan er því tilvalin til að afhenda vinnuveitanda þegar sótt er um vinnu í framtíðinni. Mikilvægt er því að hanna vefsíðuna eins snyrtilega og hægt er.

Sniðmát

Við hönnunina á vefsíðunni var forrituð html skrá sem er svo komið á netið með hjálp Git og Github.com. Til einföldunar var notað fallegt sniðmát sem fundið var á vefsíðunni html5up.net og kallast Astral. Í möppunni sem fylgir sniðmátinu má finna html skrá sem inniheldur þann kóða sem þarf að endurrita að mínum kröfum.


Brackets

Til þess að hanna síðuna að að mínum kröfum var html kóðinn settur upp í forritunarviðmótið brackets. Brackets er open-source textaritill sem auðveldar uppsetningu þegar unnið er að vefsíðum með html forritunartungumálið. Hægt er að ýta á takka efst uppi í hægra horninu til þess að sjá hvernig síðan lítur út.


Síðan sjálf

Þar sem að þetta er mín fyrsta vefsíðugerð og notkun html tók það mig nokkurn tíma að ná tökum á tungumálinu. En mér tókst þó að hanna vefsíðuna nokkurn veginn eins og ég hafði hugsað mér. Eftir að hafa skoðað síður úr fyrri áföngum fannst mér erfitt að vafra í gegnum síðurnar og óljóst hvernig hægt er að komast á milli síða. Því fannst mér mikilvægt að hafa síðuna mína eins einfalda og mögulegt er. Ég tel að mér hafi tekist það ágætlega þar sem síðan inniheldur einungis fjóra augljósa flipa: Forsíðu, ferilskrá, verkefni, hafa samband og facebook.
Mér fannst einnig mikilvægt að hægt væri jafn snyrtileg á síma sem og tölvu. Það er einungis mögulegt með því að skala allar myndir sem settar eru á síðuna á réttan máta. Sem betur fer reyndist það ekki mikið mál þar sem hönnuður sniðmátsins hafði það greinilega einnig í huga og var núþegar búinn að setja upp flokka sem ég gat svo notað til þess að setja mínar myndir inn á síðuna.


Síðunni komið í loftið

Til þess að koma vefsíðunni í loftið var notast við Git og Github.com. Settur var upp aðgangur að Github og Git halað niður af netinu. Eftir að hafa stofnað aðgang að Github var búið til nýtt repository á síðunni þar sem öll helstu gögn eru geymd:


Git er svo notað til að færa öll skjöl frá hörðum disk á tölvu og upp í skýið á Github.com. Git Bash er sett upp í möppunni sem inniheldur öll skjölin sem ætlað er upp í skýið.


Framkvæma þarf svo nokkrar skipanir í Git til þess að koma skjölunum upp í ský á Github. Ég framkvæmdi eftirfarandi skipanir af skipanalínu í eftirfarandi röð:

1. git init
2. git add .
3. git commit -m "first commit"
4. git branch -M main
5. git remote add origin https://github.com/halldorgauti/vefsida.git
6. git push -u origin main


Eftir að framkvæma þessar skipanir eru öll nauðsynleg gögn komin í skýið og það eina sem eftir er eftir er að koma síðunni í loftið. Það er gert með því að fara í settings á github og velja Branch: main undir Source. Þá hefur síðunni verið komið í loftið á url: https://halldorgauti.github.io/vefsida


Atom

Eftir að hafa byrjað að skrifa Java kóða í textaritlinum Atom komst ég að því að hægt væri að tengja þann ritil við Git og Github. Það gerir það að verkum að hægt er að hlaða upp skrám beint frá Atom textaritlinum upp í Github á mun einfaldari hátt án þess að þurfa að opna Git og slá inn óljósar skipanir á skipanalínu í hvert einasta skipti sem síðan er uppfærð. Ég hef því tamið mér héðan í frá að uppfæra síðuna í Atom og hlaða skránum þaðan beint inná Github. Atom ritillinn er þæginlegur og ekkert ósvipaður Brackets. Hægt er velja milli földa pakka til að sérsníða forritunarviðmótið að eigin þörfum.
